# 사용방법

1. [reference/refactor.md](reference/refactor.md) 를 참고하여 현 root 내 폴더구조 및 문서 구조를 개선한다. 또한 현재 작성되어있는 문서들을 refactor.md 에 정의된 규칙에 따라 변경하고 위치를 이동시킨다.

2. 해당 참고 문서에는 아래와 같은 정보가 들어있다. 그러니 반드시 [reference/refactor.md](reference/refactor.md) 를 참고하여 작업을 수행한다.

- Source/Raw/Concept/Map/Work 5종의 역할 정리
- 공통 YAML 최소 필드
- 타입별 YAML 스키마
- status enum 표준
- article unread/reading/read/archived 운영 규칙
- 파일명 규칙
- 분류 기준
- promotion / split 규칙
- Claude code 에 바로 넣을 수 있는 command prompt

3. 문서의 분류에 있어서 애매한 부분이 있으면 반드시 물어보도록 한다.

### 처음 전체 구조 변경 시

refactor.md 에 정의된 규칙에 따라 전체 변경한다.

### 한번 변경 이후 새로운 문서에 한하여 정리할 떄

새롭게 정의되고 생성된 문서들에 한하여 refactor.md 에 정의된 규칙에 따라 변경하고 위치를 이동시킨다.
